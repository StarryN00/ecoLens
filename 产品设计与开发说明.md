## 一、产品概述

### 1.1 产品定位

面向城市绿化管理部门的 **无人机航拍 + AI自动检测** 系统。无人机拍完照上传后，系统自动完成香樟树识别、虫巢检测、重叠去重和报告生成，全程无需人工判读。

### 1.2 核心用户

| 角色 | 使用场景 | 核心需求 |
|------|---------|---------|
| 无人机操作员 | 现场航拍、上传数据 | 操作简单，只需拍照上传 |
| 巡检管理员 | 查看检测结果、分配处置任务 | 清晰的虫巢分布图、一键导出报告 |
| 管理层 | 宏观决策 | 区域统计、趋势分析、预警概览 |

### 1.3 产品边界

**系统负责：**

- 接收无人机上传的航拍图片
- 自动执行：树种识别 → 虫巢检测 → GPS去重 → 报告生成
- 提供 Web 管理后台查看结果、导出报告

**系统不负责：**

- 无人机飞行控制（使用 DJI 等现有航线规划工具）
- AI模型的训练与标注（独立项目，另有文档）
- 虫害治理的具体执行
- 无人机硬件采购与维护

### 1.4 前置依赖

本系统依赖以下已完成的前置工作（由独立的标注与训练项目交付）：

| 交付物 | 说明 |
|--------|------|
| 虫巢检测模型权重 | YOLOv8 训练好的 .pt 文件 |
| 树种识别模型权重 | DeepLabV3+ 训练好的模型文件 |
| 模型使用说明 | 输入格式、输出格式、推理参数 |

### 模块3：GPS去重引擎

**职责**：将多张重叠图片中检测到的同一虫巢合并为一个，输出去重后的唯一虫巢列表。

#### 为什么需要去重

无人机航拍设置 70–80% 的航向重叠率，同一个虫巢会出现在 3–5 张相邻图片中。不去重的话虫巢数量会被严重高估。

#### 处理流程

```
raw_nest_detections (同一任务的所有原始检测)
   │
   ▼
┌──────────────────┐
│ 1. 像素→GPS反算   │  利用 EXIF 中的 GPS、高度、焦距
│                  │  将 bbox 中心像素坐标转为地理坐标
└────────┬─────────┘
         ▼
┌──────────────────┐
│ 2. DBSCAN空间聚类 │  eps=3米, min_samples=1
│                  │  距离度量: Haversine 球面距离
└────────┬─────────┘
         ▼
┌──────────────────┐
│ 3. 聚合输出       │  每个簇 → 一个唯一虫巢
│                  │  GPS 取平均值, 置信度取最高
│                  │  严重程度取最高级, 记录来源图片
└────────┬─────────┘
         ▼
    写入 unique_nests 表
```

#### GPS坐标反算逻辑

```python
import math

def pixel_to_gps(pixel_x, pixel_y, image_width, image_height,
                 photo_lat, photo_lon, altitude, focal_length, sensor_width):
    """
    将图像上的像素坐标转换为GPS地理坐标

    参数:
        pixel_x, pixel_y: 像素坐标 (0~1 相对值)
        image_width, image_height: 图片尺寸（像素）
        photo_lat, photo_lon: 拍摄点GPS坐标
        altitude: 航拍高度（米）
        focal_length: 焦距（毫米）
        sensor_width: 传感器宽度（毫米）
    """
    # 地面分辨率 (米/像素)
    gsd = (altitude * sensor_width) / (focal_length * image_width)

    # 像素偏移量（相对图片中心）
    dx_pixels = (pixel_x - 0.5) * image_width
    dy_pixels = (pixel_y - 0.5) * image_height

    # 转为地面距离（米）
    dx_meters = dx_pixels * gsd
    dy_meters = dy_pixels * gsd

    # 转为经纬度偏移
    delta_lat = dy_meters / 111320
    delta_lon = dx_meters / (111320 * math.cos(math.radians(photo_lat)))

    nest_lat = photo_lat - delta_lat  # 图片y轴向下，纬度向上
    nest_lon = photo_lon + delta_lon

    return nest_lat, nest_lon
```

#### DBSCAN去重逻辑

```python
from sklearn.cluster import DBSCAN
import numpy as np

def deduplicate_nests(detections, eps_meters=3, min_samples=1):
    """
    基于地理坐标的虫巢去重

    参数:
        detections: [{lat, lon, confidence, severity, image_id}, ...]
        eps_meters: 聚类半径（米）
    """
    if not detections:
        return []

    coords = np.array([[d['lat'], d['lon']] for d in detections])
    coords_rad = np.radians(coords)

    # eps 转为弧度: eps_meters / 地球半径
    eps_rad = eps_meters / 6371000

    clustering = DBSCAN(
        eps=eps_rad,
        min_samples=min_samples,
        metric='haversine'
    ).fit(coords_rad)

    # 按簇聚合
    unique_nests = []
    for cluster_id in set(clustering.labels_):
        mask = clustering.labels_ == cluster_id
        cluster_dets = [d for d, m in zip(detections, mask) if m]

        unique_nests.append({
            'latitude': np.mean([d['lat'] for d in cluster_dets]),
            'longitude': np.mean([d['lon'] for d in cluster_dets]),
            'confidence': max(d['confidence'] for d in cluster_dets),
            'severity': max(
                [d['severity'] for d in cluster_dets],
                key=lambda s: {'light': 1, 'medium': 2, 'severe': 3}.get(s, 0)
            ),
            'source_images': list(set(d['image_id'] for d in cluster_dets)),
            'detection_count': len(cluster_dets),
        })

    return unique_nests
```

#### DBSCAN参数说明

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| eps（聚类半径） | 3米 | 同一虫巢在不同图片中的GPS偏差通常在2–5米内 |
| min_samples | 1 | 单次检测也保留，避免漏检 |
| 距离度量 | Haversine | 球面距离，适合经纬度坐标 |

#### 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 同一棵树上多个虫巢 | 地理距离超过 eps 则识别为独立虫巢 |
| 虫巢在图片边缘被截断 | 相邻图片中的另一半检测结果会被聚类合并 |
| GPS精度不足 | 普通GPS约±2米可工作；RTK可提升至±2cm |
| 相邻树的虫巢 | 不同树上的虫巢只要距离超过 eps 就不会误合并 |

## 七、数据流转全览

```
无人机拍摄（仅拍照 + 上传）
    │
    ▼
[上传图片] ──→ 对象存储 (OSS/MinIO)
    │
    ▼
[EXIF解析] ──→ images 表 (GPS、焦距、高度、传感器参数)
    │
    ▼
[任务入队] ──→ Celery 异步队列
    │
    ▼
[预处理] ──→ 白平衡 → 切片 (640×640)
    │
    ▼
[阶段1: 树种识别] ──→ 有香樟树?
    │                      │
    │ 有                   │ 无 → image_detections(has_camphor_tree=false)
    ▼
[阶段2: 虫巢检测] ──→ raw_nest_detections 表
    │
    ▼
[像素→GPS反算] ──→ 填充 geo_latitude, geo_longitude
    │
    ▼
[DBSCAN空间聚类] ──→ unique_nests 表 (去重后)
    │
    ▼
[更新任务状态] ──→ inspection_tasks(status=completed)
    │
    ├──→ [报告生成] ──→ PDF / Excel
    │
    └──→ [预警通知] ──→ 中度以上推送邮件/短信
```

## 八、非功能性需求

| 维度 | 要求 |
|------|------|
| 性能 | 单张图片推理 ≤ 200ms (GPU)；1000张图全流程 ≤ 30分钟 |
| 并发 | 支持同时处理 3 个巡检任务 |
| 存储 | 单次巡检约1000张图 × 1MB ≈ 1GB，需支持 TB 级扩展 |
| 可用性 | 服务可用性 ≥ 99.5%；失败任务自动重试 |
| 安全 | HTTPS 通信；RBAC 权限控制；图片数据加密存储 |
| 可扩展 | 模型可热更新；去重参数可配置；未来可扩展多害虫检测 |
